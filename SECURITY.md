# Security Policy

We take security seriously in the CanIReach project. This document outlines how to report security issues and how we handle them.

---

## 🛡️ Supported Versions

We actively maintain the latest stable developer build. Version increments are tracked on the master/main branch:

| Version | Supported | Notes |
|---|---|---|
| Latest master | Yes | Active Development & Bug Fixes |

---

## 🔒 Reporting a Vulnerability

If you discover a security issue or vulnerability in CanIReach:

1. **Do Not Disclose Publicly**: Please avoid opening public issues for sensitive security vulnerabilities (such as potential remote code execution or certificate validation bypasses).
2. **Private Notification**: Contact the repository owner directly or utilize GitHub's private vulnerability reporting feature on the repository home page if enabled.
3. **Log Sanitization**: When sharing diagnostic results, screenshots, or logs, always verify that they do not contain your personal VPN configurations, SOCKS5 proxy credentials, authorization headers, or private IP address spaces.
